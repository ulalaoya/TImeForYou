# TimeForYou — מפרט אפליקציה 📅💛

> **"את רגועה, הם שמחים"** — אפליקציית יומן להזמנת שירותי בייביסיטר של רוני (בת 11).
> רוני שומרת על ילדים קטנים **כשההורים בבית** — היא מעסיקה את הילדים ומפנה להורים זמן לעצמם.

---

## 1. תמונה כללית

| נושא | החלטה |
|---|---|
| סוג המוצר | אתר PWA (מותקן על מסך הבית, בלי חנות אפליקציות) |
| אירוח | GitHub Pages (חינם) |
| מסד נתונים משותף | Firebase Firestore — מסלול Spark חינמי |
| שפה וכיווניות | עברית, RTL מלא, mobile-first |
| מייל תזכורת | **לא בגרסה 1.** שדה המייל נאסף ונשמר לעתיד |
| חפיפת הזמנות | ילד אחד בלבד בכל משבצת (משבצת תפוסה נעלמת מהזמינות) |
| הגנה על מסך רוני | קוד PIN בן 4 ספרות |
| מחיר | 20 ₪ לשעה (10 ₪ לכל משבצת של חצי שעה) |
| אזור זמן | Asia/Jerusalem |

### שני צדדים לאפליקציה
1. **צד ההורה** — רישום חד-פעמי של ילד/ה, קביעת תור ביומן, מסך סיכום עם מחיר, צפייה בתורים שלי, ביטול תור.
2. **צד רוני (מוגן PIN)** — יומן שבועי עם כל ההזמנות (אווטאר + שם + צבע הילד), רשימת ביטולים שדורשים אישור צפייה, רשימת כל ההזמנות.

## 2. חוקי היומן (לוגיקה עסקית — לדייק!)

- יומן **שבועי**, מציג **ימים א׳–ה׳ בלבד** (ניווט שבוע קדימה/אחורה; ברירת מחדל — השבוע הנוכחי).
- רזולוציה: **משבצות של 30 דקות**.
- שעות פעילות רגילות: **8:00–16:00** (משבצת אחרונה מתחילה ב-15:30).
- **חריג קיץ:** בין התאריכים **12.7.2026 עד 23.7.2026 (כולל)** היומן פתוח **רק 14:00–16:00**. משבצות מחוץ לטווח מוצגות כלא-זמינות (אפור/נעול) או לא מוצגות כלל — לבחירת העיצוב, העיקר שאי אפשר להזמין אותן.
- הזמנה = רצף משבצות באותו יום (בחירת שעת התחלה + משך בכפולות של חצי שעה, מינימום חצי שעה). אין הזמנה שנחתכת על-ידי משבצת תפוסה.
- אי אפשר להזמין משבצות שכבר עברו (לפי שעון ישראל).
- **מניעת התנגשות:** יצירת הזמנה נעשית בטרנזקציית Firestore שבודקת שכל המשבצות עדיין פנויות; אם מישהו הקדים — הודעה ידידותית ("אופס, מישהו הספיק לפנייך 🙈") ורענון היומן.
- **מחיר:** `מספר משבצות × 10 ₪`. מוצג במסך הסיכום לפני האישור וגם אחרי.

### מחזור חיים של הזמנה
```
booked ──(ההורה מבטל)──▶ cancelled ──(רוני מאשרת שראתה)──▶ cancelled_seen
```
- `booked` — מוצג ביומן של רוני בצבע הילד, עם אווטאר + שם.
- `cancelled` — המשבצות משתחררות מיד להזמנה חדשה, אבל הביטול **נשאר מוצג** ביומן של רוני (מסומן כמבוטל — למשל קו חוצה + אייקון) עד שרוני לוחצת "ראיתי ✔". יש גם רשימת ביטולים ממתינים במסך של רוני.
- `cancelled_seen` — נעלם מהיומן של רוני; נשמר בהיסטוריה.
- אם משבצת שוחררה ע"י ביטול והוזמנה מחדש ע"י ילד אחר, שני הדברים יכולים להופיע ביומן רוני באותה משבצת (הביטול מסומן כמבוטל) — לטפל בזה ויזואלית.

## 3. צד ההורה — מסכים וזרימות

### 3.1 רישום חד-פעמי (Onboarding)
טופס ידידותי, שלב-אחר-שלב אם אפשר:
- שם הילד/ה *(חובה)*
- שם ההורה *(חובה)*
- כתובת (לאן רוני מגיעה) *(חובה)*
- **בחירת אווטאר** מתוך גלריית האווטארים שבקובץ הנכסים (תינוקות וילדים קטנים, בנים ובנות מכל המגוון) *(חובה)*
- טלפון *(חובה — משמש גם כמזהה של המשפחה)*
- מייל (לתזכורות בעתיד) *(רשות)*
- הערות (אלרגיות, דברים שחשוב לדעת) *(רשות)*
- **צבע מיוחד** — *לא מוצג ולא נבחר בטופס הרישום.* המערכת מקצה לכל ילד/ה צבע באופן אקראי מפלטה בזמן הרישום, ומבטיחה שצבע לא יחזור על עצמו כל עוד יש צבעים פנויים בפלטה. הצבע משמש **רק את היומן של רוני** לצביעת ההזמנות של הילד/ה.

לאחר הרישום — פרטי המשפחה נשמרים גם ב-Firestore וגם ב-localStorage של המכשיר, כך שההורה לא נדרש להירשם שוב. אם ה-localStorage נמחק — מסך "כניסה" שמזהה לפי מספר טלפון.

### 3.2 קביעת תור
1. יומן שבועי (א׳–ה׳) עם משבצות פנויות/תפוסות. תפוס = לא ניתן ללחיצה.
2. ההורה בוחר יום, שעת התחלה ומשך (בכפולות של 30 דק׳).
3. **מסך סיכום לפני אישור:** תאריך, יום, שעות, משך, ופירוט מחיר — `X שעות × 20 ₪ = סה"כ Y ₪` (כולל חצאי שעות, למשל "שעה וחצי = 30 ₪").
4. אישור → שמירה בטרנזקציה → מסך הצלחה חגיגי 🎉 עם הסיכום והסכום.

### 3.3 התורים שלי
- רשימת תורים עתידיים (ועבר — מקופל/משני).
- כפתור ביטול על כל תור עתידי + דיאלוג אישור ("בטוח לבטל?").

## 4. צד רוני — מסכים

### 4.1 כניסה
- מסך PIN בן 4 ספרות (נשמר במסמך config ב-Firestore; ברירת מחדל תוגדר בהקמה וניתנת לשינוי ידני במסד). זו הגנת פרטיות בסיסית, לא אבטחה בנקאית — מספיק לצורך הזה.

### 4.2 היומן של רוני
- אותו יומן שבועי, אבל **מציג הכל**: כל הזמנה בצבע של הילד + אווטאר + שם הילד.
- הזמנות מבוטלות מוצגות מסומנות (קו חוצה / אפרוריות / אייקון ביטול) עם כפתור **"ראיתי ✔"** — לחיצה מעבירה ל-cancelled_seen ומעלימה מהיומן.
- באדג' התראה (מספר ביטולים שממתינים לאישור) בניווט.

### 4.3 רשימת הזמנות
- כל ההזמנות הקרובות ממוינות לפי זמן, עם פרטי ההורה (שם, טלפון — לחיץ לחיוג, כתובת, הערות) והסכום.
- סקשן ביטולים ממתינים לאישור.

## 6. ארכיטקטורה טכנית

### Stack
- **Frontend:** Vite + React, CSS פשוט או Tailwind — לבחירת המבצע, העיקר עקביות.
- **PWA:** manifest.webmanifest (שם, אייקונים מהלוגו, RTL, theme color), service worker בסיסי (cache לנכסים סטטיים). מטרה: "הוסף למסך הבית" עובד יפה באנדרואיד ו-iOS.
- **DB:** Firebase Firestore (Web SDK v10 modular). בלי Firebase Auth בגרסה 1 (זיהוי לפי טלפון + PIN לרוני).
- **Deploy:** GitHub Actions → GitHub Pages (לזכור `base` נכון ב-vite.config, ופתרון ל-SPA routing או שימוש ב-hash router).

### מבנה נתונים (Firestore)
```
config/app            { roniPin: "1234", pricePerHourILS: 20,
                        regularHours: {start:"08:00", end:"16:00"},
                        specialPeriods: [{from:"2026-07-12", to:"2026-07-23",
                                          start:"14:00", end:"16:00"}] }

families/{familyId}   { childName, parentName, address, phone, email?,
                        notes?, avatarId, colorId, createdAt }

bookings/{bookingId}  { familyId, date:"YYYY-MM-DD", startTime:"HH:mm",
                        endTime:"HH:mm", slotCount, priceILS,
                        status: "booked"|"cancelled"|"cancelled_seen",
                        createdAt, cancelledAt?, seenAt?
                        // דה-נורמליזציה לתצוגה מהירה:
                        childName, avatarId, colorId }
```
- שעות הפעילות והתקופה המיוחדת ב-config — כדי שאפשר יהיה לשנות בלי גרסה חדשה.
- כללי אבטחה (Security Rules): קריאה/כתיבה פתוחות לאוסף bookings ו-families עם ולידציית שדות בסיסית; config לקריאה בלבד מהקליינט. לתעד שזה מודל אמון פשוט המתאים לשירות שכונתי.

### קבצי פרויקט צפויים
```

/scripts/slice-assets.py  (סקריפט חיתוך חד-פעמי)

/.github/workflows/deploy.yml
SPEC.md  PROMPT.md  README.md (הוראות הקמת Firebase + deploy, בעברית פשוטה)
```

## 7. שלבי עבודה ובקרת איכות

**מודל העבודה:** התכנון והבקרה — **Fable 5** (הסשן הראשי). הביצוע — **Opus 4.8** (sub-agents / החלפת מודל). אחרי כל שלב, המתכנן בודק את התוצר מול המפרט לפני מעבר הלאה.

| שלב | תוצר | בדיקת קבלה |
|---|---|---|
| 1. נכסים | חיתוך הגיליון לקבצים | פתיחה ויזואלית של מדגם חיתוכים — נקיים, ממורכזים |
| 2. שלד | Vite+React+PWA+Firebase מחוברים, ניווט RTL | האתר עולה, כתיבה/קריאה מ-Firestore עובדת |
| 3. רישום הורה | טופס + בחירת אווטאר + שמירה | משפחה נשמרת, מזוהה בכניסה חוזרת |
| 4. יומן והזמנה | יומן שבועי, חוקי שעות, סיכום ומחיר | כל חוקי סעיף 2 — כולל חריג הקיץ ומניעת כפילות |
| 5. התורים שלי + ביטול | רשימה + ביטול | ביטול משחרר משבצות ומופיע אצל רוני |
| 6. מסך רוני | PIN, יומן צבעוני, אישור ביטולים | "ראיתי" מעלים ביטול; באדג' מתעדכן |
| 7. ליטוש + Deploy | PWA מותקן, GitHub Pages חי | התקנה על טלפון אמיתי, זרימה מלאה end-to-end |

### תרחישי בדיקה חובה (E2E)
1. הורה חדש נרשם → קובע שעה וחצי ביום ג׳ 10:00 → רואה סיכום 30 ₪ → מאשר.
2. הורה שני מנסה לקבוע 10:30 באותו יום → המשבצות תפוסות.
3. הורה ראשון מבטל → המשבצות משתחררות → אצל רוני מופיע ביטול מסומן → "ראיתי" מעלים אותו.
4. בתאריך בתוך 12–23.7.2026 אי אפשר לבחור 9:00; אפשר 14:00–15:30 בלבד.
5. שישי/שבת לא מוצגים; משבצות עבר נעולות.
6. רענון דפדפן אצל ההורה — עדיין מזוהה (localStorage).
7. PIN שגוי לא מכניס למסך רוני.
